# EduForge 项目总结与规划

> 更新时间：2026-02-15

---

## 一、产品定位

**EduForge — 开源教育 AI 引擎，让每个学生都有 AI 老师。**

| 维度 | 定位 |
|------|------|
| 一句话 | 教育界的 OpenClaw，插件化 AI 教学平台 |
| 目标市场 | 中国三四线城市 K12 教育（初中为主） |
| 核心差异化 | 开源可私部署 + AI 原生 + 插件生态 |
| 目标用户 | 学校/培训机构、独立教师、学生、家长 |
| 商业模式 | 开源免费 + 增值服务（插件市场、AI 用量、私有部署支持） |

### 竞品对比

| 维度 | EduForge | 猿题库/作业帮 | 学校自建系统 |
|------|----------|---------------|-------------|
| 部署方式 | 私有化/云端 | 仅云端 | 私有化 |
| AI 能力 | 深度内置 | 有但封闭 | 无 |
| 定制化 | 插件系统，高度可定制 | 不可定制 | 定制开发成本高 |
| 数据归属 | 学校自有 | 平台所有 | 学校自有 |
| 成本 | 低（开源） | 按人收费 | 高（开发维护） |
| 适合对象 | 注重数据安全、想定制的学校 | 大众用户 | 有 IT 团队的大校 |

## 二、当前进度

### 2.1 技术架构 ✅

```
┌─────────────────────────────────────────┐
│                 Nginx (80)              │
│           ┌────────┬────────┐           │
│           │ Web    │  API   │           │
│           │ (3000) │ (3001) │           │
│           └───┬────┴───┬────┘           │
│               │        │                │
│         Next.js 15   Fastify            │
│         React/TW     TypeScript         │
│                        │                │
│              ┌─────────┼─────────┐      │
│              │         │         │      │
│           插件引擎   AI网关   认证系统   │
│              │         │         │      │
│         ┌────┴────┐    │    JWT+bcrypt  │
│         │ 题库    │    │                │
│         │ 作业    │ DeepSeek            │
│         │ AI批改  │ 通义千问            │
│         └─────────┘ Ollama(本地)        │
│                        │                │
│              PostgreSQL (5432)           │
└─────────────────────────────────────────┘
```

- **技术栈**：Next.js 15 + Fastify + Prisma + PostgreSQL + Tailwind
- **代码量**：3,600+ 行 TypeScript
- **数据库**：20 张表
- **部署**：Docker Compose 一键启动

### 2.2 已完成功能

| 模块 | 功能 | 状态 |
|------|------|------|
| **核心引擎** | 插件加载器 + 事件总线 + 配置管理 | ✅ |
| **认证系统** | 邮箱注册/登录、JWT、角色区分（教师/学生/管理员） | ✅ |
| **AI 网关** | 统一接口、支持 DeepSeek/通义/OpenAI/Ollama、用量记录 | ✅ |
| **题库插件** | CRUD、按学科/难度/题型筛选、82 道预置题目 | ✅ |
| **作业插件** | 布置作业（选题+选班+截止时间）、学生提交、状态管理 | ✅ |
| **AI 批改插件** | 自动批改、评分+解析+评语、错题提取 | ✅ |
| **前端页面** | 10+ 页面，教师/学生导航区分 | ✅ |
| **Dashboard** | 真实数据统计卡片、最近动态 | ✅ |
| **作业管理** | 教师布置+查看提交、学生做题+查看批改结果 | ✅ |
| **错题本** | 错题列表、学生答案 vs 正确答案、AI 解析 | ✅ |
| **学习进度** | 统计卡片、学科成绩、最近成绩 | ✅ |
| **学情分析** | 班级概览、学生排名、作业完成情况 | ✅ |
| **AI 设置** | 提供商选择、模型配置、真实连接测试 | ✅ |
| **Docker 部署** | Dockerfile + docker-compose + nginx + 启动脚本 | ✅ |

### 2.3 端到端流程验证 ✅

```
教师登录 → 布置作业（选题+选班） → 学生登录 → 提交作业 → DeepSeek AI 自动批改
→ 返回 100 分 + 详细解析 → 学生查看批改结果 → 教师查看班级数据
```

**全流程已跑通，AI 批改响应正常。**

## 三、差距分析（对照产品定位）

### 🔴 关键缺失（影响核心定位）

| 缺失 | 影响 | 优先级 |
|------|------|--------|
| **微信登录/小程序** | 三四线城市用户几乎都用微信，没有就没有用户 | P0 |
| **学校管理后台** | 多学校/多租户能力，目前只有一个测试学校 | P0 |
| **教材版本适配** | 人教版/北师大版等，题目需关联教材 | P1 |
| **AI 自动出题** | 目前题库靠手动，需要 AI 根据知识点自动生成 | P1 |

### 🟡 体验不足（影响使用意愿）

| 缺失 | 影响 | 优先级 |
|------|------|--------|
| **知识点图谱** | 无法做薄弱点分析和个性化推荐 | P1 |
| **移动端适配** | 三四线城市用手机为主，PC 端不够 | P1 |
| **家长端** | 查看学情报告，这是家长付费的入口 | P2 |
| **通知推送** | 作业布置/批改完成后推送微信消息 | P2 |
| **离线支持** | 乡镇网络不稳，需要基本离线能力 | P3 |

### 🟢 已有但待完善

| 项目 | 现状 | 待完善 |
|------|------|--------|
| 题库 | 82 道 | 至少需要 500+ 道，覆盖完整知识点 |
| 错题本 | 基础列表 | 需加"重做"功能和知识点归类 |
| 学情分析 | 班级维度 | 需加知识点维度和时间趋势 |
| 部署 | Docker 配置完成 | 未实际部署到云服务器 |

## 四、接下来的工作规划

### Phase 1：最小可用产品 MVP（2 周）

> 目标：能给一个真实的初中老师试用

| 任务 | 预计时间 | 说明 |
|------|----------|------|
| 微信扫码登录 | 2 天 | 接微信开放平台，或用微信小程序 code 换 token |
| 题库扩充到 500+ | 1 天 | 用 DeepSeek 批量生成，覆盖七八九年级主要知识点 |
| 教材版本关联 | 1 天 | 人教版为主，题目关联年级+单元+知识点 |
| 移动端响应式 | 2 天 | Tailwind 响应式改造，手机端能用 |
| 部署到云服务器 | 1 天 | 买个轻量云（腾讯/阿里），docker compose 部署 |
| 基础域名+HTTPS | 0.5 天 | 备案 or 用海外域名 |
| **找 1-3 个老师试用** | 持续 | 这是最重要的——收集反馈 |

### Phase 2：产品打磨（2-4 周）

> 目标：根据试用反馈迭代，准备小规模推广

| 任务 | 说明 |
|------|------|
| AI 自动出题 | 教师选知识点+难度，AI 批量生成题目 |
| 知识点图谱 | 可视化知识点关系，标记薄弱环节 |
| 家长端（微信小程序） | 查看学情报告，这是变现的关键入口 |
| 错题重做 | 错题本核心功能，间隔复习 |
| 推送通知 | 微信模板消息：作业布置、批改完成 |
| 数据导出 | 成绩单 Excel 导出，学情报告 PDF |

### Phase 3：商业化探索（1-2 月）

> 目标：验证付费意愿，找到 PMF

| 方向 | 模式 |
|------|------|
| **SaaS 订阅** | 学校/机构按年付费，基础版免费 |
| **AI 用量计费** | 免费额度 + 超出按量付费 |
| **插件市场** | 第三方开发者卖插件，平台抽成 |
| **家长增值** | 个性化学情报告、AI 辅导，家长付费 |

### Phase 4：生态建设（长期）

- 开源社区运营（GitHub Star、贡献者激励）
- 插件市场上线
- 合作学校拓展
- 教育局/区域合作

## 五、关键指标

### MVP 阶段（Phase 1）目标

| 指标 | 目标值 |
|------|--------|
| 试用教师 | ≥ 3 人 |
| 活跃学生 | ≥ 30 人 |
| AI 批改作业数 | ≥ 100 份 |
| 题库量 | ≥ 500 道 |
| 用户反馈 | ≥ 10 条有效反馈 |

### 判断是否继续的标准

- ✅ 教师愿意持续使用（周活跃 ≥ 3 天）
- ✅ 学生完成率 ≥ 60%
- ✅ 至少 1 个老师主动推荐给同事
- ❌ 如果 1 个月内没有自然增长 → pivot 或调整

## 六、风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
| 老师不愿意用新工具 | 高 | 从最简单的功能切入（只做 AI 批改），降低迁移成本 |
| AI 批改质量不稳定 | 中 | 教师可人工复核，逐步建立信任 |
| 三四线城市付费意愿低 | 高 | 先免费获客，从家长端找付费点 |
| 政策风险（AI 进校园） | 中 | 强调"辅助工具"定位，不替代教师 |
| 技术维护成本 | 中 | 保持架构简洁，一个人能维护 |
